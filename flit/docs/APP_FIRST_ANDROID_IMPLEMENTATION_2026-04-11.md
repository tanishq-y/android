# Flit App-First + Android Shell Implementation Log (2026-04-11)

## Objective

Implement all requested steps together:

1. Android app shell for Flit with native bridge hooks.
2. Remove legacy extension-first code from active frontend paths.
3. Keep backend/frontend runnable for LAN phone testing.
4. Document every change and command.

## High-Level Outcome

1. `/connect` now uses app-first workflow and backend token-vault sessions.
2. Android shell is generated with Capacitor (`android/` folder) and native bridge classes.
3. Legacy extension-first page and unused extension hooks are removed from active code.
4. API calls are now mobile-safe via configurable API base URL.

## Native Bridge Contract

The web app expects `window.FlitNativeApp` with:

1. `openPlatformLogin(platformId, loginUrl)`
2. `exportPlatformSession(platformId)`

Android injects this object in `MainActivity` using a JS interface.

## Commands Executed

1. `npm --prefix .\fdemo\flit install @capacitor/core @capacitor/android --legacy-peer-deps`
2. `npm --prefix .\fdemo\flit install -D @capacitor/cli --legacy-peer-deps`
3. `Push-Location .\fdemo\flit; npx cap init Flit com.flit.app --web-dir=dist; Pop-Location`
4. `Push-Location .\fdemo\flit; $env:CI='1'; npx cap add android; Pop-Location`

## How To Run (LAN + Backend)

1. Backend:
   - `npm --prefix .\fdemo\flit run server`
2. Frontend on LAN:
   - `npm --prefix .\fdemo\flit run dev -- --host 0.0.0.0 --port 5173`
3. Open on phone (same Wi-Fi):
   - `http://<your-lan-ip>:5173`

## How To Run Android Shell

1. Build and sync web assets into Android project:
   - `npm --prefix .\fdemo\flit run android:sync`
2. Open Android project:
   - `npm --prefix .\fdemo\flit run android:open`
3. For physical device API connectivity set before build:
   - `VITE_API_BASE_URL=http://<your-lan-ip>:3001`

## Notes

1. `10.0.2.2` fallback is used for Android emulator when no API base is configured.
2. Physical devices should use `VITE_API_BASE_URL` (or set `localStorage.flit_api_base_url`) to reach backend.
3. Legacy extension folders were removed from the repository to keep the project app-only.
